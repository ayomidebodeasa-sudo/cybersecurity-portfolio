# Ransomware Investigation and SOC Response

## Project Summary

This project simulates a SOC investigation into ransomware activity. The aim was to investigate suspicious activity, document findings, create detection logic, organise indicators of compromise, and produce a professional incident response report.

## Objective

The objective was to understand how a SOC team investigates ransomware incidents from detection to reporting and recovery planning.

## Tools Used

- Splunk
- YARA
- MISP
- MITRE ATT&CK
- Incident response documentation

## Key Activities

- Analysed ransomware-related logs and indicators.
- Wrote Splunk queries to support investigation and threat hunting.
- Created YARA rules for identifying suspicious or malicious files.
- Organised indicators of compromise in MISP.
- Mapped findings to relevant MITRE ATT&CK techniques.
- Prepared an incident response report and recovery plan.

## Skills Demonstrated

- SOC investigation workflow
- SIEM log analysis
- Ransomware response
- Threat intelligence handling
- YARA rule creation
- IOC documentation
- Incident response reporting

This project helped me understand how SOC analysts investigate ransomware incidents, document evidence, and support business recovery after a cyberattack.
